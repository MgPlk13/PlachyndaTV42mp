# MyMasters2025

## Cyber Defense for DBMS (Flask + PostgreSQL + Analytics + Machine Learning)

Веб-застосунок для забезпечення функціональної стійкості кіберзахисту
систем керування базами даних (СКБД).  
Проєкт реалізує перевірку SQL-запитів, виявлення потенційних атак,
логування подій, аналітику та використання машинного навчання
для підвищення рівня безпеки.

---

## Мета проєкту

Метою є розробка програмної платформи, яка дозволяє:

- аналізувати SQL-запити в режимі реального часу;
- виявляти потенційно небезпечні або шкідливі запити;
- фіксувати та аналізувати інциденти безпеки;
- застосовувати ML-модель для підвищення точності виявлення атак.

---

## Функціонал

- перевірка та виконання SQL-запитів у тестовій БД;
- rule-based фільтрація небезпечних SQL-команд;
- ML-класифікація SQL-запитів (safe / malicious);
- логування подій у окремій аналітичній БД;
- аналітика атак (по годинах, останні інциденти);
- симуляція DoS-навантаження з live-графіком (SSE);
- генерація датасетів для навчання ML-моделі;
- перенавчання ML-моделі на основі зібраних логів.

---

## AI Support

Система використовує попередньо навчену модель машинного навчання
(Logistic Regression + TF-IDF) для класифікації SQL-запитів.

Запит може бути заблокований:

- за правилами (rule-based аналіз);
- або за прогнозом ML-моделі при перевищенні порогового значення ризику.

---

## Бази даних

Проєкт працює з **двома окремими базами даних**.

### Основна база даних (PostgreSQL)

Використовується для:

- виконання SQL-запитів користувача;
- тестування коректності та безпеки запитів;
- симуляції нормальної та шкідливої активності.

Тип БД: PostgreSQL  
(наприклад, навчальна база **Pagila** або будь-яка інша).

Підключення виконується:

- через файл `config_db.py`,
- або динамічно через веб-інтерфейс.

---

### База аналітики (Analytics, PostgreSQL)

Окрема база, призначена для кіберзахисту та аналізу подій.

Зберігає:

- логи SQL-запитів (allowed / blocked);
- типи атак;
- результати роботи ML-моделі.

Використовується для:

- побудови статистики та графіків;
- аналізу інцидентів;
- перенавчання ML-моделі.

# Інструкція з запуску застосунку

Нижче наведено повну покрокову інструкцію з встановлення та запуску
веб-застосунку в локальному середовищі.

---

## Крок 1. Підготовка середовища

Перед початком роботи необхідно встановити:

- Python версії 3.10 або новішої;
- PostgreSQL версії 12 або новішої;
- pip (менеджер пакетів Python).

Переконатися, що PostgreSQL запущений та доступний.

---

## Крок 2. Отримання проєкту

Клонувати репозиторій та перейти до кореневої директорії проєкту:

```bash
git clone https://github.com/MgPlk13/PlachyndaTV42mp.git
cd mymasters2025
```

---

## Крок 3. Створення та активація віртуального середовища

Створити віртуальне середовище Python:

```
python -m venv .venv
```

---

## Крок 4. Встановлення залежностей

Встановити всі необхідні бібліотеки з файлу requirements.txt:

```
pip install -r requirements.txt
```

---

## Крок 5. Налаштування баз даних

Створити дві бази даних PostgreSQL:

основну базу даних (для виконання SQL-запитів);

аналітичну базу даних analytics.

Налаштувати параметри підключення до баз даних у файлі config_db.py
або через відповідні поля веб-інтерфейсу (якщо використовується
динамічне підключення).

---

## Крок 6. Запуск застосунку

Запустити веб-застосунок командою:

```
python app.py
```

Після запуску застосунок буде доступний за адресою:

```
http://127.0.0.1:5000
```

---

## Крок 7. Робота із застосунком

    Після запуску застосунку доступні такі можливості:
    
    введення та перевірка SQL-запитів;
    
    автоматичне виявлення небезпечних SQL-конструкцій;
    
    перегляд логів дозволених та заблокованих запитів;
    
    аналітика інцидентів безпеки;
    
    симуляція навантаження (DoS);
    
    навчання та перенавчання ML-моделі.

Застосунок готовий до використання.
